## 1. 核心问题与解决方案

### 1.1 问题陈述
**工程师在处理 CAD 模型时面临的痛点：**

- ❌ STEP/IGES 文件是文本格式，难以直观理解几何结构
- ❌ CAD 软件虽然能显示模型，但难以查看底层实体信息
- ❌ 当模型出现问题（如面片缺失、拓扑错误），难以快速定位到原始文件中的具体实体
- ❌ 缺乏从"可视化"到"文件实体"的双向追溯能力

### 1.2 解决方案
**一个可视化交互式的 Brep 模型探索工具，提供：**

✅ **3D 可视化** - 直观显示模型  
✅ **交互式选择** - 点击选中面/边/顶点  
✅ **信息展示** - 显示拓扑和几何详细信息  
✅ **反向追溯** - 从可视化元素追溯到 STEP/IGES 原始实体定义  
✅ **快速排查** - 帮助工程师快速定位和解决问题

### 1.3 目标用户
**主要用户**：CAD 工程师、几何建模工程师、CAD 软件开发者

**典型使用场景**：
1. 检查 STEP/IGES 文件的拓扑结构是否正确
2. 排查模型导入/导出问题
3. 理解复杂模型的几何构成
4. 验证 CAD 数据转换的正确性
5. 学习和理解 Brep 数据结构

### 2.2 核心功能详细说明

#### 功能 1: 文件导入与解析 [P0 - 必须]

**输入**：STEP (.stp, .step) 或 IGES (.igs, .iges) 文件

**处理流程**：
1. 解析原始文件，保留所有实体信息（包括实体 ID、类型、参数）
2. 构建 Brep 拓扑结构（Shell → Face → Wire → Edge → Vertex）
3. 提取几何信息（曲面类型、曲线类型、参数域等）
4. **建立映射关系**：
   - 可视化元素 ID → 原始文件实体 ID
   - 原始文件行号 → 实体定义
   - 拓扑元素 → 几何定义

**输出**：
- Brep 模型对象（用于可视化）
- 原始文件结构树（用于追溯）
- 双向映射索引

**关键技术点**：
- 使用 OpenCASCADE 的 `STEPControl_Reader` / `IGESControl_Reader`
- 保留 `TopoDS_Shape` 的标签（Label）信息
- 使用 OCAF (Open CASCADE Application Framework) 存储映射关系

#### 功能 2: 3D 可视化与交互选择 [P0 - 必须]

**可视化需求**：
- 显示完整的 3D 模型
- 支持旋转、缩放、平移
- 支持不同渲��模式（实体、线框、着色）
- 性能要求：中等复杂度模型（1000 面）流畅渲染（>30 FPS）

**交互选择需求**：
- 点击选中：面（Face）、边（Edge）、顶点（Vertex）
- 高亮显示：选中元素用不同颜色高亮
- 多选支持：Ctrl + 点击多选
- 选择反馈：实时显示选中元素的基本信息

**技术方案**：
- **方案 A（推荐）**：Three.js + Web 界面
  - 优点：跨平台、易部署、丰富的交互库
  - 缺点：需要将 Brep 转换为网格
  
- **方案 B**：Qt + VTK/OpenGL
  - 优点：性能好、原生桌面体验
  - 缺点：部署复杂、跨平台适配工作量大

#### 功能 3: 拓扑信息展示 [P0 - 必须]

**当用户选中一个元素时，显示：**

**面（Face）信息**：
- 面 ID / 名称
- 曲面类型（平面、圆柱面、球面、B样条曲面等）
- 曲面参数（U/V 参数域）
- 面积
- 法向量
- 包含的边数量
- **追溯信息**：对应的 STEP/IGES 实体 ID

**边（Edge）信息**：
- 边 ID / 名称
- 曲线类型（直线、圆弧、B样条曲线等）
- 曲线参数
- 长度
- 起点/终点坐标
- 相邻的面
- **追溯信息**：对应的 STEP/IGES 实体 ID

**顶点（Vertex）信息**：
- 顶点 ID / 名称
- 坐标 (X, Y, Z)
- 相邻的边
- **追溯信息**：对应的 STEP/IGES 实体 ID

#### 功能 4: 反向追溯到原始文件 [P0 - 必须] ⭐核心功能

**追溯能力**：

1. **从可视化到文件实体**：
   - 用户在 3D 视图中选中一个面
   - 系统显示该面对应的 STEP 实体，例如：
     ```
     #142 = ADVANCED_FACE(...)
     #143 = PLANE(...)
     #144 = AXIS2_PLACEMENT_3D(...)
     ```

2. **显示原始文件片段**：
   - 在界面中显示相关的 STEP/IGES 文本定义
   - 高亮显示当前选中实体
   - 显示依赖关系（引用的其他实体）

3. **文件树视图**：
   - 左侧显示 STEP/IGES 文件的实体树
   - 点击实体，在 3D 视图中高亮对应的几何元素
   - 支持搜索和过滤

**实现关键**：
```python
# 伪代码示例
class BrepExplorer:
    def __init__(self, step_file):
        # 解析 STEP 文件
        self.reader = STEPControl_Reader()
        self.reader.ReadFile(step_file)
        self.reader.TransferRoots()
        
        # 构建映射
        self.shape_to_entity = {}  # TopoDS_Shape -> STEP Entity ID
        self.entity_to_shape = {}  # STEP Entity ID -> TopoDS_Shape
        self.entity_to_text = {}   # STEP Entity ID -> 原始文本
        
        self._build_mapping()
    
    def get_entity_info(self, selected_face):
        """从选中的面获取 STEP 实体信息"""
        entity_id = self.shape_to_entity[selected_face]
        entity_text = self.entity_to_text[entity_id]
        return {
            'entity_id': entity_id,
            'entity_text': entity_text,
            'dependencies': self._get_dependencies(entity_id)
        }
```

#### 功能 5: 几何计算与验证 [P1 - 重要]

**提供的计算功能**：
- 面积/体积计算
- 距离测量（点到点、点到面、面到面）
- 角度测量
- 拓扑验证（检查模型是否有效）
- 几何验证（检查曲面连续性、间隙等）

#### 功能 6: 导出与报告 [P2 - 期望]

**导出功能**：
- 导出选中元素的详细信息（JSON/CSV）
- 导出拓扑结构图（Mermaid/Graphviz）
- 生成检查报告（PDF/HTML）

## 3. 用户交互流程

### 3.1 典型工作流程

**场景 1：检查模型拓扑结构**
```
1. 用户打开 STEP 文件
2. 系统解析并显示 3D 模型
3. 用户点击一个面
4. 右侧面板显示：
   - 面的几何信息（类型、参数、面积）
   - 对应的 STEP 实体 #142 = ADVANCED_FACE(...)
   - 原始文件文本片段
5. 用户点击"查看依赖"
6. 系统高亮显示该面引用的所有实体
```

**场景 2：排查导入问题**
```
1. 用户发现模型缺少一个面
2. 在文件树中浏览所有 ADVANCED_FACE 实体
3. 找到可疑的实体 #256
4. 点击该实体，3D 视图中高亮（或显示"无法显示"）
5. 查看该实体的定义和依赖关系
6. 发现引用的曲面��义��问题
7. 定位到具体的 STEP 文件行号
```
